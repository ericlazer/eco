Quick Overview

Mizu is a revolutionary unified liquidity layer that enables seamless DeFi interactions across different virtual machines (EVM, MoveVM, SolanaVM) without bridges or wrapped assets.

Built on MoveVM with cross-VM execution capabilities, Mizu allows developers to access liquidity from any supported blockchain while users enjoy native asset trading without traditional bridging risks.

Key Features

  • Cross-VM liquidity access
  • No bridges or wrapped assets
  • Native asset settlement
  • Unified liquidity pools
  • VM-agnostic development
  • Atomic cross-VM swaps

How to Use Mizu

Prerequisites

  • Understanding of different VMs
  • Multi-chain wallet setup
  • Developer skills for integration
  • Testnet tokens for testing

Getting Started

  1. 1

    Connect Multi-VM Wallet

    Use Mizu's universal wallet connector supporting EVM, Move, and Solana wallets.

  2. 2

    Select Assets

    Choose assets from any supported VM - they appear as native, not wrapped.

  3. 3

    Execute Cross-VM Swap

    Swap between assets on different VMs in a single atomic transaction.

  4. 4

    Receive Native Assets

    Receive assets directly on their native chain without any wrapping.

Tips & Best Practices

  • Start with testnet to understand the flow
  • Keep gas on all connected chains
  • Use Mizu for arbitrage opportunities
  • Monitor cross-VM pool incentives

Frequently Asked Questions

How is this different from bridges?

Mizu doesn't lock and mint wrapped assets. It enables native cross-VM execution through cryptographic proofs.

Which chains are supported?

Currently Movement, Ethereum, BSC, Solana on testnet. More chains coming for mainnet.
